# AlphaPulse Frontend Specification

**Version**: 1.0
**Date**: 2026-01-15
**Status**: Draft

## 1. Overview

The AlphaPulse frontend is a React-based Single Page Application (SPA) designed to provide a professional MLOps dashboard for quantitative trading. It visualizes market data, trading signals generated by ML models, and system health metrics.

**Tech Stack**:
- **Framework**: React 18 (Vite)
- **UI Library**: Material UI (MUI) v5/v7
- **State Management**: Redux Toolkit + React Query (TanStack Query)
- **Routing**: React Router v6
- **Charts**: Recharts
- **HTTP Client**: Axios

---

## 2. Page Structure & Features

### 2.1. Authentication (Public)
- **Login Page**:
  - Fields: Username, Password.
  - Action: POST `/api/auth/login`.
  - Feature: Store JWT in localStorage/cookie.
- **Register Page**:
  - Fields: Username, Email, Password.
  - Action: POST `/api/auth/register`.

### 2.2. Dashboard (Private - Home)
**Goal**: High-level overview of the system and market.

- **Metrics Cards (Top Row)**:
  - System Health (Green/Red badge based on `/api/health`).
  - Latest BTC Price & 24h Change.
  - Total Signals Generated (24h).
  - Model Accuracy (Placeholder/Mock data until available).
- **Charts**:
  - **Price Trend**: Line chart showing BTC price over the last 24h/7d.
  - **Signal Confidence**: Bar chart showing confidence distribution of recent signals.
- **Recent Activity**:
  - List of the 5 most recent trading signals.

### 2.3. Market Data (Private)
**Goal**: Deep dive into historical price data and technical indicators.

- **Candlestick Chart**:
  - Interactive chart showing OHLCV data.
  - Overlays: SMA, EMA (selectable via toggle).
  - Zoom/Pan functionality (Recharts implementation).
- **Data Grid**:
  - Sortable/Filterable table of raw price data.
  - Columns: Timestamp, Open, High, Low, Close, Volume.

### 2.4. Signals Monitor (Private)
**Goal**: Monitor real-time trading signals from the ML model.

- **Filter Bar**:
  - Filter by Signal Type (BUY, SELL, HOLD).
  - Filter by Minimum Confidence (Slider: 0.0 - 1.0).
- **Signals Table**:
  - Columns: Timestamp, Symbol, Type (Color-coded), Confidence, Price at Signal.
  - Highlight high-confidence signals (> 0.8).

### 2.5. System Status (Private)
**Goal**: DevOps monitoring for the platform.

- **Service Health**:
  - Status of API, Database, MLflow, Airflow.
- **Resource Usage**:
  - CPU/Memory usage metrics (from `/api/health/metrics`).
- **Quick Links**:
  - Direct links to MLflow UI and Airflow UI (external tabs).

## 3. UI/UX Guidelines

- **Theme**: Dark Mode default (Professional Trading Terminal look).
- **Colors**:
  - Primary: Deep Blue/Purple.
  - Success (Buy/Up): Green (`#00C853`).
  - Error (Sell/Down): Red (`#D50000`).
  - Warning: Orange/Amber.
- **Responsiveness**: Fully responsive for Desktop and Tablet. (Mobile optimized as secondary priority).
- **Error Handling**: Global Snackbar/Toast notifications for API errors.

